# German/Deutsch

Dieses Dokument erläutert die Markenpolitik für das Projekt buddymon.

## Was diese Richtlinie abdeckt

Diese Richtlinie betrifft:

- Die **Projekt-Identität** des offiziellen Repositories unter github.com/Marcel-Bich/buddymon
- Das Urheberrecht an selbst erstellten Markenelementen (zukünftig z.B. ein eigenes Logo)
- Community-Normen zur Vermeidung von Verwechslung mit dem offiziellen Projekt

**Der Projektname "buddymon" selbst ist nicht markenrechtlich geschützt.** Andere Projekte dürfen den Namen verwenden, solange sie nicht vorgeben, die offizielle Version zu sein oder in direkter Konkurrenz zu diesem Projekt stehen.

## Code-Lizenz vs. Markenrechte

Der Quellcode steht unter der Buddymon License v1. Das bedeutet, dass Sie den Code zu den Bedingungen und innerhalb der Grenzen dieser Lizenz nutzen dürfen.

Diese Richtlinie betrifft jedoch einen separaten Bereich: die Projekt-Identität und die Nutzung selbst erstellter Markenelemente. Diese sind unabhängig vom Quellcode und der Softwarelizenz zu betrachten.

## Zulässige Nutzungen (kennzeichenmäßige faire Nutzung)

Sie dürfen den Projektnamen ohne Genehmigung in folgender Weise verwenden:

- Wahrheitsgemäße Aussagen wie "basierend auf buddymon" oder "Fork von buddymon"
- Verweise in Dokumentationen, Artikeln oder Diskussionen über dieses Projekt
- Quellenangaben in abgeleiteten Werken, wie von der Projektlizenz verlangt
- Verlinkungen auf dieses Repository
- Eigenständige Projekte, die den Namen "buddymon" in einem anderen Kontext verwenden, solange keine Verwechslung mit dem offiziellen Projekt entsteht

## Unzulässige Nutzungen

Sie dürfen den Projektnamen oder die Markenelemente nicht in einer Weise verwenden, die:

- eine Befürwortung, Unterstützung oder Zertifizierung durch dieses Projekt suggeriert
- eine offizielle Verbindung zu diesem Projekt suggeriert, obwohl keine besteht
- Verwirrung darüber stiftet, ob Ihr Projekt die offizielle Version ist
- den Eindruck erweckt, Ihr abgeleitetes Werk sei eine offizielle Veröffentlichung

## Domains, soziale Medien und App Stores

Registrieren Sie nicht:

- Domainnamen, die dem Projektnamen in verwechselbarer Weise ähneln und den Eindruck erwecken, die offizielle Seite zu sein
- Kennungen in sozialen Medien, die mit offiziellen Konten verwechselt werden könnten
- App-Store-Einträge, die eine offizielle Verbindung suggerieren

## Genehmigungsanfragen

Falls Sie ein Anwendungsszenario haben, das nicht von dieser Richtlinie abgedeckt ist, oder eine ausdrückliche Genehmigung für eine bestimmte Nutzung wünschen, wenden Sie sich bitte an die Projektbetreuung, indem Sie ein GitHub Issue mit dem Betreff "Trademark Permission Request" eröffnen.

Geben Sie in Ihrer Anfrage an:

- die geplante Nutzung
- den Kontext und den Zweck
- wie Sie sicherstellen wollen, dass Ihre Nutzung nicht als offizielle Unterstützung erscheint

# English/Englisch

> **Important:** This project does not claim registered trademark rights on the word "buddymon". The following rules describe community norms and copyright protection on self-created branding assets (such as a future custom logo). They exist to avoid confusion about the official version and to prevent bad-faith impersonation.

This document explains the trademark policy for the buddymon project.

## What This Policy Covers

This policy concerns:

- The **project identity** of the official repository at github.com/Marcel-Bich/buddymon
- Copyright on self-created branding assets (such as a future custom logo)
- Community norms to avoid confusion with the official project

**The project name "buddymon" itself is not a registered trademark.** Other projects may use the name, as long as they do not pretend to be the official version or directly compete with this project.

## Code License vs. Trademark Rights

The source code is licensed under the Buddymon License v1. This means you may use the code under the terms and within the limits set by that license.

This policy concerns a separate area: the project identity and the use of self-created branding assets. These are independent from the source code and software license.

## Permitted Uses (Nominative Fair Use)

You may use the project name in the following ways without permission:

- Truthful statements like "based on buddymon" or "fork of buddymon"
- References in documentation, articles, or discussions about this project
- Attribution in derivative works as required by the project License
- Linking to this repository
- Independent projects that use the name "buddymon" in a different context, as long as there is no confusion with the official project

## Prohibited Uses

You may not use the project name or branding in ways that:

- Suggest endorsement, sponsorship, or certification by this project
- Suggest official affiliation with this project when none exists
- Create confusion about whether your project is the official version
- Imply that your derivative work is an official release

## Requirements for Forks and Derivative Works

If you fork or create a derivative work:

1. Include a clear statement that your project is **not the official version** (e.g. "unofficial fork" or "not affiliated with Marcel-Bich/buddymon")
2. Do not use official branding assets from `assets/` that the project maintainer has explicitly marked as official branding (such as a future custom logo)
3. A different name is not strictly required, but is recommended if your fork pursues substantially different goals
4. Ensure your fork is not in direct commercial competition with the official project (this is already covered by the Buddymon License v1)

## Domains, Social Media, and App Stores

Do not register:

- Domain names that are confusingly similar to the project name and suggest an official affiliation
- Social media handles that could be mistaken for official accounts
- App store listings that suggest official affiliation

## Branding Assets

**Current status:** The project does not yet have a final custom logo or official branding assets.

The file `assets/buddymon-preview.gif` is a placeholder banner used for promotional purposes and is **not** covered by this policy. It will likely be replaced by a custom-made logo (TODO-un0fgo create custom repo/product logo).

Once an official logo or other official branding assets are created, they will be listed here and automatically protected by copyright and the rules of this policy.

## Permission Requests

If you have a use case not covered by this policy or want explicit permission for a specific use, contact the project maintainer by opening a GitHub Issue with the subject "Trademark Permission Request".

Include in your request:

- Your intended use
- Context and purpose
- How you plan to make clear that your use is not an official endorsement

But please, if you find this useful, be cool about it. Don't pretend to be the official project. Don't clone the repo and try to sell it as your own. Don't use my branding to mislead people.
